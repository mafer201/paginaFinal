INTRODUCCIÓN AL MODELAMIENTO DE SOFTWARE



1.- ¿Qué es Software?

El software se refiere a un conjunto de programas, datos, procesos y pautas que permiten realizar diversas tareas en un sistema informático. Es la parte intangible de un sistema informático que abarca desde los programas del dispositivo informático hasta el sistema operativo, las aplicaciones y otros componentes relacionados con la informática. Sin software, estos dispositivos son inútiles. Por ejemplo, no podríamos navegar por Internet ni utilizar nuestros ordenadores de forma eficaz sin un navegador de Internet o un sistema operativo.

2.- ¿Qué es el modelado de Software?

El modelado de software es una técnica utilizada para tratar con la complejidad inherente a los sistemas de software. Consiste en crear modelos que ayudan a los ingenieros de software a visualizar y especificar la estructura y el comportamiento del sistema a construir. Estos modelos también facilitan la comunicación con los clientes y el equipo de desarrollo.

Algunos ejemplos de modelado de Software:

  • microsoft Windows: Permite al usuario manejar y e interactuar con los distintos segmentos de un computador, mediante un entorno de usuario amigable, basado en ventanas y representación visual.
  • Mozilla Firefox: Un navegador de Internet sumamente popular, disponible para descarga sin pago. Conecta al usuario con la World Wide Web, para realizar búsquedas de datos y otros tipos de operaciones virtuales.
  • Google Chrome: Otro navegador de Internet, ofrecido por la empresa Google, cuya ligereza y velocidad lo hizo rápidamente popular entre los usuarios de Internet.

3.- ¿Qué es un modelo en el desarrollo de software?

Un modelo en el desarrollo de software es una metodología o enfoque utilizado para organizar y guiar el proceso de desarrollo de software. Estos modelos establecen el orden en el que se realizarán las actividades y proporcionan requisitos de entrada y salida para cada una de ellas.

  • Modelo en cascada: Este modelo ordena rigurosamente las etapas del desarrollo del software, de modo que el inicio de una etapa de desarrollo debe esperar el fin de la etapa anterior.
  • Modelo de prototipos: Este modelo pertenece a los modelos evolutivos y se basa en la construcción rápida de prototipos con el uso mínimo de recursos.
  • Modelo en V: Este modelo divide el proceso de desarrollo en tres partes principales: diseño, implementación y pruebas de integración y cualificación.
  • Modelo RAD: Este modelo se utiliza para proyectos con requisitos predefinidos y se enfoca en el desarrollo rápido de aplicaciones.

4.- ¿Para que se modela un software?

En el desarrollo de software siempre es importante utilizar un modelo, ya que es lo que permite conocer los elementos concretos de una empresa, rompe con las barreras de comunicación y ayuda a identificar los puntos de mejora.

A continuación, te presento algunas razones clave por las cuales se modela el software antes de comenzar a programar:

  • Nos ayuda a visualizar un sistema como deseamos que sea diseñado.
  • Nos da una plantilla que guía el proceso de construcción.
  • Documenta las decisiones que hemos tomado.
  • Los modelos son herramientas que apoyan al proyecto a largo plazo.

    • 5.- ¿Cuáles son las fases del modelado de Software?

    • Definición de requisitos:El primer paso fundamental en las etapas del desarrollo de software es comprender y definir claramente los requisitos del cliente.
    • Análisis y diseño:El análisis implica descomponer los requisitos en componentes más pequeños y comprender cómo se interrelacionan. Luego, el diseño se enfoca en definir la arquitectura del sistema.
    • Implementación:La implementación implica la escritura de programas, la creación de bases de datos, la integración de componentes y la realización de pruebas unitarias.
    • Pruebas:Las pruebas son esenciales para identificar errores, verificar la precisión del software y asegurar que cumpla con los requisitos establecidos.
    • Despliegue: El despliegue puede implicar la instalación del software en servidores, la configuración de la infraestructura necesaria y la capacitación de los usuarios finales.
    • Mantenimiento:El mantenimiento del software es una etapa crucial que implica garantizar el rendimiento continuo, solucionar problemas y realizar mejoras adicionales según sea necesario.

    6.- ¿Qué es arquitectura de software?

    Es un concepto que se refiere a la planificación fundamentada en patrones, modelos y abstracciones teóricas, cuando es necesario desarrollar un proyecto de software con algún grado de complejidad. Es el paso previo a la implementación.

  • Tipos de arquitectura de software:
  • Patron cliente-servidor
  • Patron de capas.
  • Patron modelo-vistas-controlador.
  • Patron broker.
  • Patron master-slave.
  • Patron cliente-servidor.

    • 7.-¿Qué es la interfaz de usuario?

    La interfaz de usuario (UI) es el puente que conecta a las personas con la tecnología. Es el medio mediante el cual controlamos aplicaciones de software o dispositivos de hardware. A través de elementos gráficos como botones, barras de desplazamiento, teclados y pantallas táctiles, los usuarios interactúan con sistemas electrónicos como ejemplo:

    • Atractivo Visual: La apariencia es prioritaria. Un diseño atractivo permite que los usuarios se sientan cómodos y se identifiquen con el programa.
    • Claridad: La información transmitida debe ser clara y concisa para evitar errores o confusiones durante la interacción con el software.
    • Coherencia: Todos los elementos de una aplicación deben mantener unidad en diseño y propósito. Esto permite a los usuarios crear patrones de uso de forma intuitiva.

    8.- ¿Cuáles son los tipos de modelado de Softwares?

  • Modelado de gestión: Se genera la información que conduce el proceso de gestión, se identifica a dónde va la información y quién la procesa.
  • Modelado de datos: Se definen los almacenes de datos y cómo se relacionan los almacenes entre sí.
  • Modelado del proceso: Se utiliza para añadir, modificar, suprimir o recuperar un objeto de datos.
  • Generación de aplicaciones: Se utiliza una herramienta de cuarta generación que permite crear el software y facilitar la construcción del programa.
  • Pruebas y entrega: El proceso de desarrollo finaliza realizando pruebas de calidad del software diseñado con la herramienta RAD, posteriormente se realiza la implementación de la aplicación.

    • 9.-¿Qué es UML?

    El Lenguaje Unificado de Modelado (UML) fue creado para forjar un lenguaje de modelado visual común y semántica y sintácticamente rico para la arquitectura, el diseño y la implementación de sistemas de software complejos, tanto en estructura como en comportamiento.

    UML cumple con los siguientes requerimientos:

    • Brindar una explicación detallada de la semántica de cada concepto de modelado UML.
    • Establecer una definición formal de un metamodelo común basado en el estándar MOF que especifique la sintaxis abstracta del UML.
    • Especificar los elementos de notación de lectura humana para representar los conceptos individuales de modelado UML.
    • Definir formas que permitan hacer que las herramientas UML cumplan con esta especificación.

    10.- ¿Qué es un programador de software?

    Un programador de software es un profesional que se encarga de escribir, diseñar, desarrollar y mantener software. El software es un conjunto de instrucciones que un ordenador puede seguir para realizar tareas específicas. Los programadores utilizan diferentes lenguajes de programación para crear aplicaciones, sitios web, sistemas operativos y otros tipos de software. Su trabajo implica resolver problemas, diseñar algoritmos y colaborar en equipos de desarrollo para crear soluciones informáticas efectivas.

    Las funciones de un programador de software:

  • Escribir código con diferentes lenguajes de programación.
  • Funcion a sistemas operativos, páginas web, aplicaciones multiplataforma.
  • Analizar y depurar código existente para optimizar.
  • Colaborar en equipos de desarrollo para crear soluciones informáticas efectivas.

    • 11.- ¿Qué es un diseñador de Software?

    Un desarrollador de software es el profesional que se encarga de diseñar, crear y elaborar software mediante el uso de lenguajes de programación. Gracias a esto, los sistemas operativos informáticos funcionan de manera eficiente y correcta. El proceso de trabajo del desarrollador implica crear nuevos sistemas pensando en las necesidades que tiene el usuario y haciendo revisiones y monitorización para controlar que todo el proyecto funcione adecuadamente.

    Funciones principales de un desarrollador de software:

    • Diseñar el código y realizar pruebas.
    • Crear el software teniendo en cuenta las necesidades del usuario.
    • Seguir las pautas que le ha indicado el cliente para el proyecto.
    • Comunicar la situación y el progreso a las personas encargadas mediante informes.

    12.- ¿Qué es un algoritmo?

    Un algoritmo es un conjunto de instrucciones o pasos específicos que permiten resolver un problema o tomar una decisión. En informática, los algoritmos son secuencias de operaciones lógicas y ordenadas que controlan procesos. Estas características son fundamentales:

  • Secuencia: Los algoritmos operan en secuencia, procesando uno a la vez, desde las primeras instrucciones hasta las últimas.
  • Precisión: Son precisos y específicos, lo que garantiza resultados coherentes.
  • Finitud: Los algoritmos son conjuntos finitos de pasos, lo que significa que siempre terminan.
  • Orden: Los pasos están organizados de manera lógica y ordenada.

    • 13.- ¿Que es la ingeniería de requerimientos?

    La ingeniería de requerimientos es el proceso de recopilar, analizar y verificar las necesidades del cliente para un sistema de software. Su objetivo es entregar una especificación de requerimientos de software correcta. En otras palabras, se trata de entender y definir de manera precisa los objetivos, funciones y restricciones del sistema que se va a construir.

    A continuación, algunas definiciones y aspectos clave:

    • La IEEE lo describe como “la condición o capacidad requerida por el usuario para resolver un problema o alcanzar un objetivo” o como “la representación documentada de una condición o capacidad” que debe satisfacer el sistema.
    • Según Zave, es la rama de la ingeniería del software que establece los objetivos, funciones y restricciones del sistema.
    • Loucopoulos lo ve como un proceso iterativo y cooperativo de análisis del problema, documentación y prueba de conocimiento adquirido.
